# DISCUSSION

## API definition
API 需要清晰无歧义，这也导致了它缺乏复用性。即不能进行重载，否则语言模型常常会错误调用 \

## Example
有时不方便用语言表述的样例，通过样例让LLM了解意图是很好的

## Model
对于不同的语言模型，效果差异很明显，GPT4能够有很强的推理能力。对于少量的example就可以推理出类似情况下的解决方案，而GPT3.5在这方面弱很多

## Hyperparameter
尽量使用低想象的参数，这样能确保生成更稳定的代码

## Human feedback
人类的二次提示，能很好的纠正LLM生成的错误代码

## Detail instruction
充满细节的任务描述，如告诉它一些关键的步骤，通常会取得更好的结果，尤其对于能力低的LLM来说。如GPT3.5，因为它难以推断出中间的操作。

## History conversation
对于新的任务，是否需要考虑之前历史任务的对话。有些是需要考虑的，如GPT3.5经常将pose理解为XYZ，忽略了转角。如果上述任务中，人类纠正过了，后续代码就几乎不会犯这样的错误。
